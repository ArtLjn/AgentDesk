## 1. 文档定位

本目录用于沉淀“基于多智能体协同的智能工单处理系统设计与实现”的毕业设计文档。项目来源于 AI Agent 学习项目，当前文档的目标不是把全部学习内容都纳入论文范围，而是把其中已经成型的多智能体工单处理系统整理为一个体量适中的本科毕设项目。

系统核心是：用户提交工单后，由多个 Agent 按职责协同完成意图理解、分类、处理、审核、人工复核、通知与追踪展示。技术实现以 FastAPI、LangGraph、React、SQLAlchemy 2.0 async、MySQL、Qdrant 和 OpenAI 兼容接口为主。

v1.3 之后，系统的人机协同闭环进一步扩展为“AI 处理 + 人工审核 + 用户补充 + 自动恢复”。当审核员判断工单缺少订单号、支付流水等关键信息时，可提交 `request_info` 决策，系统将工单置为 `waiting_user_input`，前端详情页展示补充输入框；用户补充的消息写入 `ticket_messages`，随后工作流携带沟通上下文从 `process` 节点恢复处理。

## 5. 毕设范围声明

本项目定位为本科毕业设计，不追求复杂企业级工单平台的完整能力。系统重点展示以下能力：

- 多智能体分工协同：分类、处理、审核、协调。
- 自然语言工单结构化：使用 TicketIntentAgent 提取标题、分类、优先级、影响范围等字段。
- 工作流编排：使用 LangGraph 表达工单处理状态流转。
- RAG 辅助处理：通过知识库检索增强处理 Agent 的回答质量。
- 人机协同闭环：AI 无法可靠闭环时进入人工审核队列，审核决策可恢复工作流。
- 用户补充闭环：审核员可请求用户补充关键信息，用户回复后自动恢复工单处理。
- 实时状态反馈：通过 WebSocket 展示工单执行进度。
- 可观测性：记录 trace 和 span，便于展示 Agent 执行过程。
- 前后端完整闭环：提供登录、工单提交、列表、详情、知识库、监控、审核工作台、设置和统计页面。
