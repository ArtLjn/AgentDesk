# Python 后端编码规范

## 1. 分层规范

后端代码应优先放在 `src/multi_agent_system/` 下的既有目录中：

| 目录 | 放置内容 |
| --- | --- |
| `api/` | FastAPI 应用和路由 |
| `agents/` | Agent 实现 |
| `workflow/` | LangGraph 状态机和状态定义 |
| `models/` | Pydantic 模型和枚举 |
| `tools/` | Agent 可调用工具 |
| `core/` | 基础设施能力 |

不要在项目根目录散放 Python 文件。

## 2. 类型与模型

- 请求和响应优先使用 Pydantic 模型。
- 枚举值应集中定义，避免在多处硬编码。
- 函数签名尽量标注类型。
- 对外返回的数据结构应稳定，避免随意变更字段名。

## 3. 异步规范

- FastAPI 路由优先使用 `async def`。
- 数据库访问使用异步方法。
- 后台工作流使用 `asyncio.create_task()` 时，应确保异常被捕获和记录。
- 不在异步函数中执行长时间阻塞操作。

## 4. 日志规范

- 使用项目已有日志工具，不使用 `print` 调试。
- 日志应包含工单 ID、节点名称或 trace ID 等定位信息。
- 错误日志应记录异常原因，但不输出密钥等敏感信息。

## 5. 异常处理

- API 层对用户输入错误返回 400。
- 查询不存在的资源返回 404。
- 外部服务不可用返回 503 或记录降级信息。
- 工作流异常应将工单标记为 `failed` 并写入错误字段。

## 6. 测试规范

- 新增工作流节点应补充状态流转测试。
- 新增 API 应补充接口测试。
- 新增 Agent 行为应补充成功和降级测试。
- 测试文件放在 `tests/` 对应目录下。
