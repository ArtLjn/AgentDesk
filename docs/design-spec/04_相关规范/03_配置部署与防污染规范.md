# 配置部署与防污染规范

## 1. 配置管理

系统配置由 `src/multi_agent_system/config.py` 中的 `Settings` 管理，可通过环境变量或 `.env` 覆盖。

主要配置项：

- `llm_base_url`
- `llm_api_key`
- `embedding_base_url`
- `qdrant_url`
- `qdrant_collection`
- `llm_model`
- `embedding_model`
- `max_retries`
- `review_threshold`
- `max_concurrency`

## 2. 敏感信息

- 不提交 `.env` 文件。
- 不在文档中写入真实 API Key。
- 不在日志中打印完整密钥。
- 示例配置使用占位值。

## 3. 本地启动

后端启动：

```bash
uvicorn src.multi_agent_system.api.app:app --reload
```

前端启动：

```bash
cd web
npm install
npm run dev
```

可选依赖通过 Docker Compose 启动，例如 Qdrant、Prometheus、Grafana 等。

## 4. 防污染规则

- 禁止提交 `__pycache__/`。
- 禁止提交 `.DS_Store`。
- 禁止提交 `*.pyc`。
- 禁止在项目根目录创建 `test.py`、`tmp.py`、`debug.py` 等临时脚本。
- 临时测试代码应放在 `tests/` 中。
- 不修改 `.git/` 目录内部文件。
- 删除不再使用的代码，不保留无意义 dead code。

## 5. Git 规范

提交信息使用 Conventional Commits：

- `feat:` 新功能。
- `fix:` 修复问题。
- `docs:` 文档变更。
- `refactor:` 重构。
- `test:` 测试变更。
- `chore:` 构建、依赖或维护工作。

提交前应确认：

- 没有敏感信息。
- 没有系统生成文件。
- 相关 lint 或测试已通过。
- 文档与代码变更保持一致。
