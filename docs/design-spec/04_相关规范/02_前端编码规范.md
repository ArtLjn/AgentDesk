# 前端编码规范

## 1. 目录规范

前端代码位于 `web/src/`：

| 目录 | 内容 |
| --- | --- |
| `pages/` | 页面级组件 |
| `components/` | 通用组件 |
| `components/ui/` | 基础 UI 组件 |
| `hooks/` | 自定义 hook |
| `lib/` | API 客户端和工具函数 |
| `types/` | TypeScript 类型 |

## 2. 类型规范

- 后端返回结构应在 `web/src/types/index.ts` 中定义。
- API 请求和响应应尽量使用已有类型。
- 不在页面组件中重复定义大型接口。

## 3. API 调用规范

- HTTP 请求集中放在 `web/src/lib/api.ts`。
- 页面组件不直接拼接复杂请求逻辑。
- 错误状态应给出明确提示。
- WebSocket 连接逻辑优先复用 hook。

## 4. UI 展示规范

- 管理端页面以清晰、紧凑、易扫描为主。
- 工单状态、分类、优先级应使用统一 Badge 或状态组件。
- 表格、详情、统计区域应保持一致的间距和信息层级。
- 不在页面中加入与功能无关的大段说明文字。

## 5. 状态处理规范

- 加载中状态应使用 skeleton 或明确 loading。
- 接口错误应显示可理解的错误信息。
- WebSocket 断开时不应导致页面崩溃。
- 对可能为空的字段进行兜底展示。

## 6. 毕设演示规范

演示页面应重点展示：

- 工单从提交到完成的实时变化。
- 分类和优先级识别结果。
- 处理 Agent 生成的方案。
- 审核评分。
- 执行 trace 和节点耗时。
