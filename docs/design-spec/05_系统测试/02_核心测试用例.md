## 8. 人工审核（Phase 5 补充）

### 8.1 四种触发场景

| 编号 | 场景 | 预期结果 |
| --- | --- | --- |
| TC-701 | 投诉类工单 escalate | 进入 `human_review_wait`，工单状态 `pending_human_review`，队列可见 |
| TC-702 | AI 审核失败 retry_count=3 | 进入 `human_review_wait`，trigger_type=`review_failed` |
| TC-703 | 工作流执行抛异常 | 兜底为 `error_fallback` 类型 pending 审核单 |
| TC-704 | completed 工单提交 satisfied=false | 自动创建 `user_request` 类型 pending 审核单，工单状态回退 |

### 8.2 五种决策路由

| 编号 | 输入决策 | 预期路由 |
| --- | --- | --- |
| TC-711 | `approve` | 沿用原 processing_result → notify → complete |
| TC-712 | `rewrite`（含 rewritten_result） | 覆盖 processing_result → notify → complete |
| TC-713 | `reprocess` | 清空 processing_result + retry_count → process → review |
| TC-714 | `reject` | 标记为已驳回 → complete |
| TC-715 | `request_info` | 审核单置为 decided，工单进入 `waiting_user_input`，写入审核员补充消息 |

### 8.3 队列、详情、统计接口

| 编号 | 场景 | 预期结果 |
| --- | --- | --- |
| TC-721 | `GET /reviews/queue` 支持按 trigger_type/category/priority 筛选 | 队列按优先级 + 等待时长排序 |
| TC-722 | `GET /reviews/{ticket_id}` 返回工单上下文 + 当前 + 历史审核 | 含 trace 摘要、AI 建议 |
| TC-723 | `GET /reviews/stats` 计算 ai_adoption_rate | 分子=采纳数，分母=有 AI 建议的决策数 |
| TC-724 | 决策已 decided 的工单再次提交 | 返回 409 |
| TC-725 | `POST /reviews/{ticket_id}/decision` 提交 `request_info` | 返回 `next_node=waiting_user_input`，`workflow_resumed=false` |

### 8.4 WebSocket 事件 / Trace 集成

| 编号 | 场景 | 预期结果 |
| --- | --- | --- |
| TC-731 | 工单进入待审核时连接 `/ws/monitor` | 收到 `review_requested` 事件 |
| TC-732 | 审核员提交决策 | 收到 `review_decided` 事件（含 next_node） |
| TC-733 | `GET /tickets/{id}/trace` | span 树含 `span_type=human_decision` 的 decided span |
| TC-734 | 审核员请求用户补充 | 收到 `user_input_requested` 事件和普通 `ticket_update` |

## 9. 用户补充信息（v1.3 补充）

| 编号 | 场景 | 预期结果 |
| --- | --- | --- |
| TC-801 | 查询不存在工单的沟通记录 | `GET /tickets/{id}/messages` 返回 404 |
| TC-802 | 工单处于 `waiting_user_input`，用户提交补充消息 | 写入 `ticket_messages`，随后调用 `resume_from_user_input` |
| TC-803 | 工单不处于 `waiting_user_input`，用户提交补充消息 | 返回 409，不写入消息 |
| TC-804 | 用户提交空白补充消息 | Pydantic 校验失败，返回 422 |
| TC-805 | 读取某工单沟通记录 | 按 `created_at` 升序返回审核员请求和用户回复 |
| TC-806 | 用户补充后恢复处理 | 同一 trace 中出现 `user_input_resume`、`process`、`review` span |
| TC-807 | 用户补充后 ReviewerAgent 评分通过 | 工单进入 `completed` |
| TC-808 | 用户补充后 ReviewerAgent 评分仍不通过 | 按原重试和人工审核策略继续流转 |
