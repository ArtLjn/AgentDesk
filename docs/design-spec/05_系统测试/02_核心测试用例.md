# 核心测试用例

## 1. 工单创建

| 编号 | 输入 | 预期结果 |
| --- | --- | --- |
| TC-001 | 提交有效工单内容 | 返回 `ticket_id` 和 `received` 状态 |
| TC-002 | 批量提交两条工单 | 返回两个独立结果 |
| TC-003 | 查询已存在工单 | 返回完整工单详情 |
| TC-004 | 查询不存在工单 | 返回 404 |

## 2. 分类与路由

| 编号 | 输入 | 预期结果 |
| --- | --- | --- |
| TC-101 | “系统崩溃无法登录” | 分类为技术类，进入处理流程 |
| TC-102 | “账单重复扣费” | 分类为账务类，进入处理流程 |
| TC-103 | “我要投诉服务态度” | 分类为投诉，进入升级流程 |
| TC-104 | “如何修改密码” | 分类为咨询，进入自动回复 |

## 3. 审核与重试

| 编号 | 场景 | 预期结果 |
| --- | --- | --- |
| TC-201 | 审核评分高于阈值 | 工单进入通知并完成 |
| TC-202 | 审核评分低于阈值且未超限 | 返回处理节点重试 |
| TC-203 | 重试次数达到上限 | 工单进入失败处理 |

## 4. 知识库

| 编号 | 输入 | 预期结果 |
| --- | --- | --- |
| TC-301 | 上传标题和正文完整的文档 | 返回 `chunks_added` |
| TC-302 | 缺少标题 | 返回 400 |
| TC-303 | 缺少正文 | 返回 400 |
| TC-304 | Qdrant 不可用 | 返回知识库服务不可用 |

## 5. WebSocket

| 编号 | 场景 | 预期结果 |
| --- | --- | --- |
| TC-401 | 连接单工单 WebSocket | 接收该工单节点更新 |
| TC-402 | 连接全局监控 WebSocket | 接收所有工单节点更新 |
| TC-403 | 客户端断开连接 | 后端清理连接列表 |

## 6. Trace

| 编号 | 场景 | 预期结果 |
| --- | --- | --- |
| TC-501 | 查询已处理工单 trace | 返回 trace 和 span 树 |
| TC-502 | 查询不存在 trace | 返回 404 |
| TC-503 | 查询 trace 统计 | 返回节点耗时统计 |

## 7. 统计分析

| 编号 | 场景 | 预期结果 |
| --- | --- | --- |
| TC-601 | 查询统计接口 | 返回分类分布、优先级分布、处理统计 |
| TC-602 | 提交满意度反馈 | 反馈记录写入并影响满意度统计 |
