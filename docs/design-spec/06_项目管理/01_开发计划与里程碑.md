# 开发计划与里程碑

## 1. 阶段划分

| 阶段 | 时间建议 | 主要任务 | 产出 |
| --- | --- | --- | --- |
| 阶段一：需求与设计整理 | 第 1 周 | 明确毕设范围，整理设计文档 | 设计规范、需求说明 |
| 阶段二：核心流程完善 | 第 2-3 周 | 检查工单工作流、Agent、API | 可运行后端 |
| 阶段三：前端联调 | 第 4 周 | 完善工单、知识库、监控页面 | 可演示前端 |
| 阶段四：测试与修复 | 第 5 周 | 补充测试用例，修复问题 | 测试报告 |
| 阶段五：论文撰写 | 第 6-7 周 | 撰写系统分析、设计、实现、测试章节 | 论文初稿 |
| 阶段六：答辩准备 | 第 8 周 | 准备演示脚本、PPT、常见问题 | 答辩材料 |

## 2. 当前项目基础

当前项目已经具备：

- FastAPI 后端服务。
- LangGraph 工单工作流。
- 分类、处理、审核、协调 Agent。
- SQLite 数据存储。
- Qdrant 知识库工具。
- React 前端页面。
- 部分 pytest 测试。
- Docker 部署脚本。

后续重点不是大规模新增功能，而是整理、稳定、验证和论文表达。

## 3. 优先级建议

### P0：必须完成

- 工单提交到完成的完整演示链路。
- 工单列表和详情展示。
- 分类、处理、审核、路由说明。
- 核心测试可运行。
- 设计文档和论文主体。

### P1：建议完成

- 知识库上传和 RAG 演示。
- WebSocket 实时进度展示。
- Trace 执行树展示。
- 统计分析页面。

### P2：可选完成

- Prometheus/Grafana 可视化。
- 更复杂的评价指标。
- 更丰富的知识库样例。
- Docker 一键演示优化。

## 4. 答辩演示脚本建议

1. 打开前端仪表盘，说明系统整体能力。
2. 提交咨询类工单，展示自动回复路径。
3. 提交技术类工单，展示分类、处理、审核、完成路径。
4. 提交投诉类工单，展示升级路径。
5. 打开工单详情，展示状态、处理结果和审核评分。
6. 打开 trace 或 Agent 监控，展示多 Agent 执行过程。
7. 上传知识库文档，说明 RAG 增强思路。
